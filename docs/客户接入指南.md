# flow-batch 接入指南

> 面向:造梦次元内部使用 BFF 的内容制作方(首个用户:良维「青少年模式批量内容」)
> 版本:**v2.0 · 2026-07-05**(对外字段全面重构,详见文末变更历史)

## 一、这是什么

flow-batch 是造梦次元的**内容批量生产通道** BFF。给你三个简单接口,你不用去理解造梦次元 Studio 编辑器里的复杂结构,就能:

1. **创建角色**(`POST /flow-batch/character`)
2. **一键创建作品**(`POST /flow-batch/flow`)—— 一次调用完成"新建 → 保存内容结构 → 更新元信息 → 生成初始 TTS → 内容风控 → 提审"6 步
3. **生成图片**(`POST /flow-batch/image`)—— 生图并自动转存到造梦次元可用的 CDN 域名

**紧凑接口 spec**(字段/错误码/例子):看 [`docs/api/`](./api/) 目录里的 4 份 md。

## 二、鉴权

**必须在造梦次元后台申请开通「超级创作者」角色**(`ROLE_SUP_CREATOR`)。开通后:

- 登录 https://dreama.ideaflow.pro
- 打开浏览器开发者工具,找任意一个发往 `dreama.ideaflow.pro` 的请求
- **直接复制这个请求里的** `Authorization` 和 `Uid` 两个 header 值
- 每次调 flow-batch 接口带上这两个 header

```
Authorization: metatube-<你的 JWT>
Uid: <你的 24 位长用户 id>
```

**BFF 不发独立 API Key** —— 复用你现有的造梦次元账号身份即可。

**Token 过期 / 权限不对** → BFF 返回 `4001`,通常 `details.errorCode` 是 `DREAMA_TOKEN_INVALID` → 你重新登录 dreama.ideaflow.pro 并重新复制 header。

## 三、统一响应壳

所有接口返回:

```json
{
  "code": 0,
  "message": "ok",
  "data": { /* 各接口不同 */ },
  "requestId": "req_01H...",
  "details": {
    "errorCode": "BAD_REQUEST"
  },
  "retryable": false,
  "serverTime": 1783100000000
}
```

`code === 0` 才成功。失败时 `code >= 4000`(详见 §七 错误码)。

**怎么看失败原因**:

- 先看顶层 `code`:这是大类(鉴权/参数/风控/下游/内部)
- 再看 `details.errorCode`:这是更细的原因(例如 `DREAMA_TOKEN_INVALID`、`CONTENT_REJECTED`)

## 四、接口 1 · 创建角色

**`POST /flow-batch/character`**

### 请求

```bash
curl -X POST https://tools.ideaflow.pro/flow-batch/character \
  -H "Authorization: metatube-<JWT>" \
  -H "Uid: <你的 uid>" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "阿凯",
    "gender": "男",
    "age": 45,
    "avatar_url": "https://img.ideaflow.pro/xxx.jpg",
    "banner_url": "https://img.ideaflow.pro/xxx.jpg",
    "summary": "退休软件工程师,喜欢用生活比喻教编程",
    "personality": "耐心温和,喜欢引导而不是直接给答案",
    "speech_style": "爱说\"来,我们一步一步来\",讲话慢,句尾常带笑意"
  }'
```

### 字段说明

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `name` | string | ✅ | 角色名,1-50 字 |
| `gender` | enum | ✅ | `"男"` / `"女"` / `"未知"` |
| `age` | int \| null | ✅ | 年龄;未知传 `null` |
| `avatar_url` | url | ✅ | **必须是 `img.ideaflow.pro` 域名的图**,用接口 3 生成 |
| `banner_url` | url | ✅ | 同上,形象图 |
| `summary` | string | ✅ | 一句人设简介,1-500 字 |
| `personality` | string | ✅ | 性格描述,1-500 字 |
| `speech_style` | string | ✅ | 说话习惯 / 口头禅,1-500 字 |
| `timbre_id` | string | ❌ | 音色 id;**良维当前批量场景一般不用传**,不传 BFF 会按性别自动选默认音色 |

**BFF 内部固定处理**:图片默认标记为 AI 生成(合规必需),你不用管。

**实操建议**:先别管 `timbre_id`。把人设、头像、形象图跑通更重要;只有你明确要指定音色时再补这个字段。

### 响应

```json
{
  "code": 0,
  "data": { "character_id": "000005081889730783100935" }
}
```

**注意**:`character_id` 是造梦次元后端**新分配**的 24 位雪花 id。保存这个 id,下一步建作品要用。

## 五、接口 3 · 生成图片(先讲,因为角色/作品都要图)

**`POST /flow-batch/image`**

### 请求

```bash
curl -X POST https://tools.ideaflow.pro/flow-batch/image \
  -H "Authorization: metatube-<JWT>" \
  -H "Uid: <你的 uid>" \
  -H "Content-Type: application/json" \
  -d '{
    "prompt": "45 岁老年程序员,戴老花镜,慈祥微笑,写实风格",
    "size": "2048x2048",
    "max_images": 1
  }'
```

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `prompt` | string | ✅ | 提示词,1-2000 字 |
| `size` | string | ❌ | 图片尺寸,`"宽x高"` 格式;**当前批量生产请统一用 `"2048x2048"`**。Seedream 4.5 现在非方形容易直接 400;默认 `"2048x2048"` |
| `max_images` | int | ❌ | 1-15,默认 1 |
| `reference_urls` | string[] | ❌ | 参考图 URL 数组,最多 14 张;**只有图生图才用** |
| `generation_mode` | enum | ❌ | `"single"` / `"set"`;良维当前纯文生图可忽略 |
| `call_type` | string | ❌ | 用途追踪标签(便于日志/账单统计);不影响作品内容,不传也行 |

### 响应

```json
{
  "code": 0,
  "data": {
    "image_urls": [
      "https://img.ideaflow.pro/flow-batch/<uid>/img-1783xxx-0-<ulid>.jpg"
    ],
    "elapsed_ms": 15700
  }
}
```

**关键**:URL **一定是 `img.ideaflow.pro` 域名**(BFF 已经内部做了转存)。造梦次元 Studio 只识别这个域名的图。**别拿 `tools.ideaflow.pro` 的原始生图 URL 直接给作品用**,会显示裂图。

**良维当前最小可用请求**:只传 `prompt` + `size` + `max_images` 就够了。后面 3 个可选字段先忽略。

## 六、接口 2 · 一键创建作品 ⭐

**`POST /flow-batch/flow`**

### 请求

```bash
curl -X POST https://tools.ideaflow.pro/flow-batch/flow \
  -H "Authorization: metatube-<JWT>" \
  -H "Uid: <你的 uid>" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "编程小屋",
    "cover_url": "https://img.ideaflow.pro/xxx.png",
    "summary": "放学后走进一家旧书店,遇到退休程序员",
    "main_role_id": "000005081889730783100935",
    "opening": [
      { "type": "background", "title": "背景介绍", "content": "放学后你走进了一家旧书店" },
      { "type": "narration", "content": "推开门,木门发出吱呀一声" },
      {
        "type": "role",
        "role_id": "000005081889730783100935",
        "content": "哟,小朋友来找书?",
        "user_btns": ["我想学编程", "我随便看看", "有游戏推荐吗"]
      }
    ],
    "preset_turns": [
      {
        "reply": "呀,你想学编程?好方向哦。",
        "buttons": ["从哪里开始?", "难吗?", "先看什么?"]
      },
      {
        "reply": "编程就像做菜,把事情拆成一小步一小步。",
        "buttons": ["有例子吗?", "我想试试", "太抽象"]
      }
    ],
    "story": {
      "background": "退休工程师在旧书店里教中学生理解编程,内容健康向上"
    },
    "category": "chat",
    "publish": true
  }'
```

### 字段总览

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `name` | string | ✅ | 作品名,1-100 字 |
| `cover_url` | url | ❌ | 封面 URL(**用接口 3 生成的 `img.ideaflow.pro` URL**);不传封面缺失,审核过不了 |
| `summary` | string | ❌ | 作品简介,1-500 字 |
| `main_role_id` | 24 位 string | ✅ | 主角 id(接口 1 返回的 `character_id`) |
| `supporting_role_ids` | string[] | ❌ | 配角 id 数组,最多 9 个 |
| `opening` | array | ✅ | 开场剧本,**1-10 段**(见下) |
| `preset_turns` | array | ❌ | 预设对话轮,**0-50 轮**(见下),默认 `[]` |
| `story` | object | ✅ | AI 自由对话设定(见下) |
| `category` | enum | ❌ | 当前仅支持 `"chat"`,默认 `"chat"`;`"story"` / `"game"` 待造梦次元后端补真实分类 id 后放开 |
| `publish` | bool | ❌ | 默认 `true` 自动提审;`false` 只存草稿 |

**作品所属人**:BFF 从 `Uid` header 自动兜底,你不用在 body 里重复传 `user_id`。

### `opening[i]` 三种 type

```json
{ "type": "background", "title": "背景介绍", "content": "..." }
{ "type": "narration", "content": "..." }
{
  "type": "role",
  "role_id": "...",
  "content": "角色说的话",
  "user_btns": ["按钮1", "按钮2", "..."]
}
```

- `background` — 背景交代(灰色小卡片,不带角色不带按钮)
- `narration` — 环境旁白(不带角色不带按钮)
- `role` — 角色说话
  - `role_id` 请只用本作品的 `main_role_id` 或 `supporting_role_ids`
  - 当前接口不会帮你把角色名自动映射成 id,也不会替你自动补到 `supporting_role_ids`
  - `user_btns` 是用户可点的多个选项按钮,**都进入同一路径,不做分支**
  - 如果你不需要“让用户点一下再继续”的感觉,`user_btns` 可以不传

### `preset_turns[i]`

```json
{ "reply": "角色预设回复", "buttons": ["选项1", "选项2", "..."] }
```

一轮就一句"主角预设话" + 多个用户按钮(1-20 个)。之后**进入 AI 剧情模式**(`story.background` 驱动)。

**上限 50 轮**:BFF 内部会拆批向下游申请节点 id,50 轮以内都稳。超过 50 轮 route 层直接 400。

### `story`

```json
{ "background": "剧情背景 / 世界观 / 内容尺度约束(1-2000 字)" }
```

`background` 必填。你可以把它理解成"**这个作品进入自由对话后,AI 应该遵守的世界观、语气和内容边界**"。AI 参数(温度/top_p/max_tokens)由 BFF 内部按业务场景调好默认,你不用传。

**命名提醒**:`story` 这个字段名偏技术历史,但你接入时只要把它当成"AI 自由对话设定"就行。

### 响应

```json
{
  "code": 0,
  "data": {
    "pipe_id": "000005081893077116272642",
    "publish_status": "submitted",
    "studio_url": "https://studio.ideaflow.pro/pipe.html?pipe_id=000005081893077116272642"
  }
}
```

- `publish_status`:`"submitted"` = 已提审(`publish=true` 时);`"draft"` = 只保存草稿
- **打开 `studio_url` 立刻能看到作品**,确认内容是否符合预期

## 七、错误码

| 顶层 `code` | 常见 `details.errorCode` | HTTP | 含义 | 你怎么处理 |
|---|---|---|---|---|
| `0` | - | 200 | 成功 | — |
| `4001` | `DREAMA_TOKEN_MISSING` / `DREAMA_TOKEN_INVALID` | 401 | 没带 header、登录过期、或没有超级创作者权限 | 重新登录 dreama.ideaflow.pro 后重新复制 header;仍失败就找管理员开权限 |
| `4002` | `BAD_REQUEST` / `INVALID_ARGUMENT` | 400 | 请求参数校验失败 | 看 `details.issues` 或 `message`,按字段名逐个改 |
| `4041` | `RESOURCE_NOT_FOUND` | 404 | 角色 id 不存在等 | 检查是不是把旧 id、别人的 id、或拼错的 24 位 id 传进来了 |
| `4291` | `CONTENT_REJECTED` | 400 | 内容命中造梦次元风控 | 看 `details.rejected` 里的字段列表,把命中的词句改成更中性、更青少年向的表达后重试 |
| `5021` | `UPSTREAM_LLM_FAILED` / `UPSTREAM_CYAPI_FAILED` / `UPSTREAM_STUDIO_FAILED` | 502 | 下游 cyapi / studio / llm-api 报错 | 看 `retryable`; 为 `true` 时优先直接重试,仍失败再带 `requestId` 联系我 |
| `5001` | `INTERNAL_SERVER_ERROR` | 500 | BFF 内部错 | 联系我(郑世宇),带上 `requestId` |

## 八、批量生产建议

给 50 个作品的推荐节奏:

```
for 每个作品:
    a. 调接口 3 生成头像(2048x2048)+ 形象图(2048x2048)+ 封面(2048x2048)
    b. 调接口 1 建角色(拿到 character_id)
    c. AI 生成作品结构(opening + preset_turns + story)
    d. 调接口 2 建作品(拿到 pipe_id + submitted 状态)
    e. (可选)打开 studio_url 验证一眼
```

**推荐分两段跑**:

1. **试跑阶段**:先挑 3~5 个作品,`publish=false`,人工打开 `studio_url` 看开场、角色、按钮、AI 设定都对
2. **正式阶段**:确认模板没问题后,再把同一批 JSON 改成 `publish=true` 顺序提审

**并发**:建议**单 uid 顺序跑**,不要并发(studio pipe/save 是全量保存,并发同一账号可能相互覆盖)。50 个大约 30 分钟能全部走完。

**可直接改的顺序模板**:

```bash
for 每个作品JSON:
  1. 先调 /image 生成头像
  2. 再调 /image 生成形象图
  3. 再调 /character 拿 character_id
  4. 再调 /image 生成封面
  5. 最后调 /flow
     - 首轮联调: publish=false
     - 正式提审: publish=true
  6. 保存返回的 requestId / character_id / pipe_id / studio_url
```

**幂等**:每次调用都带 `X-Request-Id` 头。BFF 目前**只记日志,不做去重**,所以重试前先看上一条到底是成功了还是失败了。

## 九、常见问题

**Q:作品建出来了但审核团队说"必要字段没过"?**

A:优先检查这 4 件事:

- `cover_url` 有没有传
- `summary` 有没有传
- `category` 有没有明确传(当前只接受 `"chat"`,别靠默认)
- `publish` 是不是误传成 `false` 了

**Q:`studio_url` 打开看到"角色显示裂图"?**

A:你的 `avatar_url` / `banner_url` / `cover_url` 一定要用**接口 3 返回的 `img.ideaflow.pro` 域名 URL**,别用其他域名(尤其别用生图上游直接返回的 tools 域名 URL)。

**Q:token 过期怎么处理?**

A:BFF 会返回 `4001`,同时 `details.errorCode` 往往是 `DREAMA_TOKEN_INVALID`。你重新登录 dreama.ideaflow.pro,重新复制 `Authorization` 和 `Uid` 即可,无需重启任何东西。

**Q:`CONTENT_REJECTED` 之后到底怎么改?**

A:别只盯着单个词,按这 3 步改最快:

1. 先看 `details.rejected` 里点名了哪几句
2. 把这几句里的恋爱暗示、身体接触、暴力伤害、惊悚猎奇、成人化表达改成更中性的成长/友情/探索表达
3. 连同前后一句一起顺一遍,避免上下文合起来还像同类内容

对青少年模式最稳的写法是:多写成长、学习、冒险、社团、任务,少写暧昧、制服诱惑、危险惩罚、身体描写。

**Q:为什么我明明按文档写了,还是在建作品时报参数错?**

A:先查这两个高频坑:

- `preset_turns` 是否超过 50 轮了
- `opening` 里的 `role_id` 是否真的是本作品的主角/配角 id

**Q:想改一个已经建好的作品?**

A:本 BFF 当前只支持"从 0 建",不支持"改已有"。改作品请到 Studio 编辑器手动改。
